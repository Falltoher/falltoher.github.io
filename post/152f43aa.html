JavaEE定制篇

JDK

安装步骤：

  1. mkdir /opt/java
  2. 通过xftp上传到/opt/java目录下
  3. cd /opt/jdk
  4. 解压tar -zxvf jdk-8u221-linux-x64.tar.gz
  5. mkdir /usr/local/java
  6. mv /opt/java/jdk1.8.0_221 /usr/local/java
  7. 配置环境变量的配置文件 vim /etc/profile
  8. export JAVA_HOME=/usr/local/java/jdk1.8.0_221
    9.export PATH=$JAVA_HOME/bin:$PATH
  9. source /etc/profile 【让新的环境变量生效】

测试：
编写一个hello.java程序，使用JAVA命令行工具，在终端上显示“Hello World!”。


使用命令：

运行时环境：将/etc/profile文件中的JAVA_HOME变量设置为/usr/local/java/jdk1.8.0_221/jre

将JAVA_HOME环境变量加入到PATH环境变量中，如果系统提供了java命
令的软链接，那么可以不用设置PATH环境变量。

Tomcat

步骤：

  1. 上传安装文件到/opt/apache-tomcat-8.5.51/bin目录下
  2. 解压缩文件到/opt/apache-tomcat-8.5.51/webapps目录下
  3. 进入解压缩目录，启动tomcat
  4. 测试tomcat是否启动成功

概述

shell是一个命令行解释器，它为用户提供了一个向Linux内核发送请求以便运行程序的界面系统级程序，用户可以用shell来启动、挂起、停止甚至是编写一些程序。

执行方式

脚本格式要求：

  1. 以#!/bin/bash开头
  2. 脚本要有可执行权限

变量

Linux shell中的变量分为系统变量和用户自定义变量，系统变量是系统定义好的，用户自定义变量是用户自己定义的变量。

shell变量的定义：
基本语法：
定义变量：变量名=变量值
撤销变量：unset 变量名
声明变量：declare -a 变量名 或 declare -x 变量名
声明静态变量：readonly 变量名，注意不能unset
查看变量：echo $变量名 或 echo ${变量名}

规则：

  1. 变量名称只能是英文字母、数字和下划线，且不能以数字开头
  2. 等号两边不能有空格
  3. 变量名称一般习惯使用大写字母，系统变量使用大写字母，用户自定义变量使用小写字母
