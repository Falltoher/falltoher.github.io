response 提供一些方法，能够很方便的进行页面信息提取

回调逻辑

前面获取到的 url 是相对路径，需要拼接成绝对路径，然后再请求这个绝对路径：
用这个url再次请求，然后指定一个回调函数，这个回调函数就是用来解析帖子详情页面的；


这里需要用yield返回并构造scary.Request对象，传入三个参数：

  • url：请求的url，下层页面的地址；
  • callback：回调函数，用来解析下层页面的数据；
    parse()函数是处理当前页面的逻辑，下层页面需要新写一个函数来处理；
    写法和parse()类似，函数名可以自定义，参数是response；
    注意：参数传入的是callback=self.post_parse，不是callback=self.post_parse()，不要加括号
  • cb_kwargs：回调函数的参数，这里是将上层页面的item传递给下层页面的回调函数；
    是一个字典类型，为了给post_parse函数传递参数，这里是将item传递给post_parse函数；
    在post_parse(self, response, **kwargs)函数中，可以通过kwargs获取到这个参数；

Pycharm Debug

（1）先在工程下随便找一个 py 文件，里面啥也不写，执行一下，然后点击右上角的【Edit Configurations】；
（2）在系统中找到 scrapy 包中的 cmdline.py 文件
（4）在 Name 里面填入调试的名字，比如 Scrapy；
（4）在 Script path 里面把 cmdline.py 的路径填进去；
（5）在 Parameters 里面填入 Scrapy 的参数：crawl bbs_spider -o bbs.csv；
（6）点击右下角的 【ok】，在主界面点【Debug】就可以进行调试了
